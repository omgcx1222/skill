# vibe-coding

- 倾向于编写清晰易读的代码，避免不必要的复杂性。
- 对于任何不明确、不了解的需求、逻辑，都需要先向开发者提问确认。
- 除非另有说明，否则不要删除原代码中开发者编写的 console 输出。
- 对于变量的命名和复杂的逻辑，必须添加中文注释。

## 在开始生成代码前，先充分阅读理解当前项目的代码结构。

## 如果新开发的需求偏离了指定的md文档，在代码生成完成后同步修改掉md文档的内容，且修改完后通知开发者。（未指定md文档请忽略此条）

## 对于已经存在的代码（包括但不限于方法、逻辑、组件），不要重复生成，如果已经存在相关的代码无法满足当前的需求，先向开发者确认：

- 修改当前代码（但不要影响旧逻辑，因为这个方法已经在其他地方被调用了）
- 直接创建新的代码（包括但不限于方法、逻辑、组件）

新创建的代码必须符合项目的代码规范和风格。对于重复性的逻辑代码、前端组件，优先考虑封装复用，避免重复编写。
